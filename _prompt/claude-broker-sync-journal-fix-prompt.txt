Claude Code Prompt — Final Fix for Broker Sync vs Journal Trade Duplication

Audit the current trading bot repo and implement the final cleanup for the broker-sync vs journal trade separation.

Context

The app now correctly separates summary-level meanings:
- Dashboard shows live broker reality
- Journal summary shows:
  - Journal Open
  - Live Positions
- broker_sync trades are excluded from performance metrics by default

However, one important inconsistency remains:

Current problem

The Journal table still shows broker_sync rows as if they are normal journal trades.

Example current state:
- Journal summary:
  - Journal Open: 0
  - Live Positions: 1 (broker-synced)
- But the Journal table also includes:
  - ETHUSD | broker_sync | open

This causes duplication and a misleading mental model:
- broker_sync is already represented by the Live Positions card
- broker_sync should not also appear in the default journal trade table as a normal trade row

Goal

Make the Journal page a clean strategy/performance view by default.

Do not show broker_sync rows in the default Journal table view.

Keep broker-sync positions visible through:
- Dashboard open positions
- Journal summary Live Positions card
- optional explicit broker-sync-only view later if needed

Required behavior

1. Journal summary stays as-is conceptually
Keep:
- Total Trades = performance trades only
- Journal Open = journal-backed open trades only
- Live Positions = broker reality
- broker_sync excluded from default performance metrics

2. Journal table default view must exclude broker_sync rows
The default Journal table should show only real strategy trades / journal-backed trades.

That means:
- rows where strategy/source is broker_sync should be filtered out from the default dataset
- open broker-sync reconciliation rows should not appear in the default Journal table

3. Do not break dashboard
Dashboard should continue to show:
- Open Positions = broker reality
- Strategy = Broker Sync for orphan broker-held positions
- stop/target/risk may remain blank for broker-sync rows

4. Do not corrupt performance metrics
Do not change win rate, total pnl, avg win/loss, total trades, or closed/open performance counts to include broker_sync by default.

Recommended implementation direction

Preferred architecture:
- Journal page default = strategy/performance ledger only
- Broker-sync positions = broker/reconciliation layer, not normal journal rows

So the fix should remove broker_sync rows from the default Journal table response or default Journal page dataset.

Implementation requirements

Backend

Inspect and update the journal list endpoint and any filtering logic used by the Journal page.

Likely files:
- src/server/routes/journal.js
- src/journal/tradeJournal.js
- any helper that builds the journal trade list response

Required backend behavior:
- default journal table dataset excludes broker_sync rows
- filtering should be explicit and shared if possible
- if there is already includeBrokerSync support, preserve it but do not make it default
- open broker_sync rows should not appear in the default trade list

If useful, support an explicit opt-in query flag such as:
- includeBrokerSync=true
but default behavior must remain false

Frontend

Inspect:
- client/src/pages/JournalPage.jsx
- client/src/services/journal.js
- any journal table component

Required frontend behavior:
- default Journal page should render only non-broker_sync rows
- do not rely only on client-side filtering if backend can provide the correct default dataset
- keep Live Positions card visible
- do not remove the Journal Open / Live Positions split

Optional enhancement

If it is easy and clean, add a future-safe path for explicit broker sync visibility, such as:
- a query param
- a hidden debug/admin mode
- a dedicated filter later

But do not add UI complexity unless needed.
The main fix is: default table view should no longer show broker_sync rows.

Tests

Update or add tests for:

Backend
- default journal list excludes broker_sync rows
- default open journal list excludes broker_sync rows
- includeBrokerSync=true includes them if supported
- performance summary behavior remains unchanged
- dashboard open positions still include broker-sync positions

Frontend
- Journal page default table does not render broker_sync rows
- summary cards still show:
  - Journal Open
  - Live Positions
- dashboard still shows Broker Sync position in open positions

Expected end state

Dashboard
- Open Positions: 1
- ETHUSD shown as Broker Sync

Journal summary
- Journal Open: 0
- Live Positions: 1 (broker-synced)

Journal table
- does NOT show broker_sync open row by default
- shows only real strategy trades
