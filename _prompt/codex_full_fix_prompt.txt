==================================================
PROJECT CONTEXT
==================================================

The trading bot is no longer supposed to behave like a fixed 15-minute worker loop.

The production model is now:

- Heroku Scheduler triggers the app every 10 minutes
- Scheduler calls POST /api/cycle/run
- The bot starts immediately
- It executes the full trading cycle as fast as required
- It updates live runtime state while running
- The dashboard reflects the running cycle in real time
- When complete, the UI shows completion and then returns to waiting for the next trigger

The goal is not a fake timer UI. The UI must be driven by real backend runtime state.
