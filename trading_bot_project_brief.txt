TRADING BOT PROJECT BRIEF (UPDATED)

OBJECTIVE
Build a complete automated trading system that can:
- analyze real-time market data (stocks + crypto)
- operate across global trading sessions
- make trading decisions based on strategy
- execute trades (paper mode initially)
- manage open positions (risk + exits)
- provide full visibility through a dashboard
- scale across a multi-asset universe (20+ crypto + stocks)

---

SYSTEM OVERVIEW

The system operates in repeating cycles (every 15 minutes) and consists of four core engines:

1. Market Scan Engine
2. Decision Engine
3. Execution Engine
4. State / Journal Engine

A frontend dashboard sits on top to visualize activity in real time.

---

SESSION-AWARE EXECUTION MODEL

Crypto:
- Trades 24/7
- Always included in every cycle

US Stocks:
- Trades only during New York session
- Also available during London/New York overlap

Session Behavior:

Tokyo → Crypto only
London → Crypto only
New York → Crypto + Stocks
London + NY Overlap → Crypto + Stocks
Off-session / Weekend → Crypto only

FULL SYSTEM FLOW

1. SYSTEM START
- Load environment configuration
- Connect to Alpaca
- Validate API + trading mode
- Load settings (risk, strategy, assets)

2. WAIT FOR CYCLE
- Align execution with 15-minute candle closes

3. START CYCLE
- Initialize cycle context
- Resolve current market session
- Record timestamp and mode

4. LOAD ACCOUNT + POSITIONS
- Fetch account data
- Fetch open positions

5. LOAD INTERNAL STATE
- Load open trades store
- Load decision history
- Load journal files

6. MONITOR OPEN POSITIONS
- Check stop loss / take profit
- Close trades if conditions met

7. BUILD MARKET UNIVERSE
- Crypto always active
- Stocks session-dependent

8. FILTER ELIGIBLE SYMBOLS
- Crypto always included
- Stocks only when NY open

9. FETCH MARKET DATA
- Retrieve 15-minute bars

10. VALIDATE DATA
- Reject insufficient data

11. COMPUTE INDICATORS
- ATR
- Breakout levels
- Volume metrics

12. RUN STRATEGY
- Approve or reject trades

13. APPLY RISK GUARDS
- Max open positions (5)
- Daily limits
- Duplicate checks

14. LOG DECISIONS
- Persist results
- Update dashboard

15. EXECUTE TRADES
- Place paper trades
- Store positions

16. UPDATE STATE
- Journal updates
- Activity feed updates

17. END CYCLE
- Log summary

18. WAIT FOR NEXT CYCLE

---

TRADE LIFECYCLE

ENTRY
- Strategy approves → trade executed

OPEN POSITION
- Track entry, stop, target, pnl

EXIT
